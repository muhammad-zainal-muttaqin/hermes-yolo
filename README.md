# Hermes-YOLO

> Autonomous hyperparameter & strategy search for TBS (Tandan Buah Segar) oil palm ripeness detection using YOLO.  
> Inspired by [Andrej Karpathy's autoresearch](https://github.com/karpathy/autoresearch).

---

## Problem

Detect 4 ripeness classes of oil palm fresh fruit bunches (TBS):

| Class | Label | Description |
|:-----:|:-----:|:------------|
| B1 | Unripe | Green, hard |
| B2 | Under-ripe | Starting to ripen |
| B3 | Ripe | Optimal harvest |
| B4 | Overripe | Past optimal |

**Core challenge**: B2/B3 confusion is high due to visual similarity. B1/B4 are rare classes.

## Key Findings

**What consistently works:**
- Top-K Ensemble: +5.1% over baseline
- Extended training (50+ epochs): better convergence
- CopyPaste semantic augmentation: helps rare B1/B4 classes
- Higher input resolution (768px): helps small B4 detection

**What doesn't work:**
- Standard TTA (Test-Time Augmentation): no gain
- Adaptive label smoothing: not the bottleneck
- Architecture swap alone: dataset quality is the constraint

---

## Dataset

- 4 classes: B1 (unripe), B2 (under-ripe), B3 (ripe), B4 (overripe)
- Sources: DAMIMAS + LONSUM plantations
- Class imbalance: B1/B4 underrepresented
- See [CONTEXT.md](CONTEXT.md) for full problem definition

## Reproducibility

All experiments use `seed=42`. To reproduce any run:

```bash
# Check experiment config
cat runs/detect/experiments/runs/BREAK_101/train/args.yaml

# Re-run with same config
python autoresearch.py --resume BREAK_101
```
